Key insight: The client is completely blocked during the ~50ms round trip, but the CPU does useful work for only ~0.1ms of that time (0.2%). The rest is waiting for bytes to travel over the wire. This idle time is why async programming exists — and why request-response alone can't power real-time features like live order tracking.
