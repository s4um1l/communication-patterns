Key insight: After the one-time upgrade handshake (~350 bytes), every chat message costs just 2-20 bytes of framing overhead. The equivalent HTTP POST would carry ~400 bytes of headers per message. For a 50-message chat session, WebSocket uses ~1,400 bytes total vs HTTP's ~35,000 bytes — a 96% reduction in overhead. The connection is full-duplex: both sides send independently, simultaneously, over a single TCP connection.
